# Envio HyperIndex Local Indexing Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Run the f(x) trader-profile data pipeline locally with Envio HyperIndex, a local RPC router fed by ChainList/free Ethereum RPCs, local Postgres, and the existing Next.js app.

**Architecture:** Keep the app, indexer, and RPC routing as separate local services. ChainList supplies candidate public RPC metadata; an internal rpc-router health-checks and rotates usable endpoints; Envio indexes f(x) contract events through that router; the Next.js app syncs normalized indexed data into app-owned Postgres tables for UI/API queries. Goldsky/The Graph remain optional hosted fallbacks, not Phase A requirements.

**Tech Stack:** Next.js 16, Docker Compose, Postgres 16, Envio HyperIndex, Hasura GraphQL generated by Envio, TypeScript, Node test runner, ChainList `https://chainlist.org/rpcs.json`, ChainID `https://chainid.network/chains/eip155-1.json`.

---

## Ground Rules

- Phase A stays local: no GHCR, no hosted indexer, no mandatory paid RPC.
- Do not expose raw public RPC URLs, Hasura admin secrets, or indexer credentials to the browser.
- Use free/public RPC endpoints only for POC reliability and failover. Respect provider rate limits; do not use rotation to abuse providers.
- No official f(x) leaderboard API is assumed. Leaderboard ingestion is a later scraper/source-adapter task.
- Keep Envio-generated tables separate from app-owned analytics tables.
- Prefer small start blocks/ranges while proving the pipeline; full historical backfills may require archive-capable paid RPC or Envio HyperSync later.

## Phase 1: Add RPC Router Service

**Objective:** Provide one internal Ethereum JSON-RPC endpoint for Envio and app workers while rotating and health-checking free RPC providers.

**Files:**
- Create: `apps/rpc-router/package.json`
- Create: `apps/rpc-router/tsconfig.json`
- Create: `apps/rpc-router/Dockerfile`
- Create: `apps/rpc-router/src/server.ts`
- Create: `apps/rpc-router/src/providerPool.ts`
- Create: `apps/rpc-router/src/chainlist.ts`
- Create: `apps/rpc-router/src/health.ts`
- Create: `apps/rpc-router/src/rateLimit.ts`
- Create: `apps/rpc-router/src/providerPool.test.ts`
- Modify: `package.json`
- Modify: `docker-compose.yml`
- Modify: `.env.example`
- Modify: `infra/self-host/.env.example`

**Implementation notes:**
- Expose internal service URL: `http://rpc-router:8545`.
- Host bind is optional for debugging only; default should be internal Compose networking.
- Read static/manual endpoints from:
  - `ETHEREUM_RPC_URL`
  - `ETHEREUM_RPC_FALLBACK_URLS`
  - `CHAINLIST_ETHEREUM_RPC_URLS`
  - optional provider-key URLs: `GOLDSKY_RPC_URL`, `CHAINSTACK_RPC_URL`, `ALCHEMY_RPC_URL`
- Fetch dynamic endpoints from:
  - preferred: `https://chainlist.org/rpcs.json`
  - fallback/canonical chain metadata: `https://chainid.network/chains/eip155-1.json`
- Cache remote endpoint lists with a TTL; never fetch ChainList on every RPC request.
- Filter endpoint candidates:
  - keep `https://` for HTTP JSON-RPC;
  - keep `wss://` only for future subscription support;
  - reject placeholders such as `${INFURA_API_KEY}` unless explicitly substituted;
  - reject localhost/private/link-local/file/unix/internal URLs from remote metadata;
  - de-duplicate normalized URLs;
  - prefer ChainList `tracking=none`, then `limited`, then `unspecified`; treat `yes` as lowest priority.
- Health-check candidates with:
  - `eth_chainId` must equal `0x1`;
  - `eth_blockNumber` must return a recent number;
  - optional quorum comparison of head block across providers.
- Circuit breaker behavior:
  - timeout per request, e.g. `RPC_TIMEOUT_MS=8000`;
  - mark provider unhealthy/cooldown on 429, 5xx, timeout, stale block, malformed JSON;
  - retry idempotent reads on the next healthy provider;
  - do not retry non-idempotent writes by default.
- Guard expensive calls:
  - cap `eth_getLogs` block range for free RPCs;
  - reject debug/trace methods for POC;
  - cap JSON-RPC batch size.

**New env vars:**
```env
RPC_ROUTER_URL=http://rpc-router:8545
CHAINLIST_RPC_SOURCE_URL=https://chainlist.org/rpcs.json
CHAINID_RPC_SOURCE_URL=https://chainid.network/chains/eip155-1.json
CHAINLIST_ETHEREUM_RPC_URLS=
RPC_TIMEOUT_MS=8000
RPC_PROVIDER_COOLDOWN_SECONDS=60
RPC_MAX_RETRIES=2
RPC_MAX_GET_LOGS_BLOCK_RANGE=2000
RPC_ROTATION_STRATEGY=weighted_round_robin
```

**Tests:**
- `providerPool` rotates healthy providers.
- 429/5xx/timeouts trigger cooldown.
- `eth_chainId` mismatch rejects provider.
- placeholder/private URLs are filtered.
- `eth_getLogs` range guard rejects oversized requests.

**Verification:**
1. Run `pnpm --filter @fx-trader-profiles/rpc-router test`.
2. Run `docker compose --env-file .env up -d --build rpc-router`.
3. Run:
   ```bash
   docker compose exec -T rpc-router node -e 'fetch("http://127.0.0.1:8545", {method:"POST", headers:{"content-type":"application/json"}, body:JSON.stringify({jsonrpc:"2.0",id:1,method:"eth_chainId",params:[]})}).then(r=>r.text()).then(console.log)'
   ```
   Expected: result `0x1`.
4. Add `rpc-router` to monitor output.

**Commit:**
```bash
git add apps/rpc-router package.json docker-compose.yml .env.example infra/self-host/.env.example scripts/self-host-monitor.sh
git commit -m "Add local Ethereum RPC router"
```
